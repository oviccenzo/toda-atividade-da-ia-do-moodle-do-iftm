Atividade Avaliativa – Aplicação Web com Inteligência Artificial para Predição de Evasão Escolar
Disciplina: Inteligência Artificial
Curso: Ciência da Computação
Modalidade: Individual
Data de entrega: 05/03/2026
1. Objetivo da Atividade

O objetivo desta atividade é desenvolver uma aplicação web em Python integrada a um modelo de Inteligência Artificial capaz de realizar classificação ou previsão de evasão de alunos, utilizando um dataset previamente fornecido.

A atividade visa consolidar os seguintes conhecimentos:

    Pipeline completo de um projeto de IA

    Treinamento e avaliação de modelos de Machine Learning ou Redes Neurais

    Salvamento e carregamento de modelos treinados

    Integração de modelos de IA em aplicações web

    Organização e versionamento de projetos com GitHub

2. Descrição da Atividade

Cada aluno deverá:

    Realizar a análise e preparação do dataset de alunos fornecido

    Definir o problema de predição (evasão ou não evasão)

    Treinar um modelo de IA adequado ao problema

    Avaliar o modelo utilizando métricas apropriadas

    Salvar o modelo treinado

    Desenvolver uma aplicação web simples em Python

    Integrar o modelo salvo à aplicação web

    Permitir que a aplicação receba dados de entrada e retorne a previsão de evasão

3. Requisitos Técnicos Mínimos

O projeto deve conter obrigatoriamente:

    Linguagem: Python

    Modelo de IA treinado (Machine Learning ou Rede Neural)

    Salvamento do modelo treinado

    Aplicação web funcional (Flask)

    Integração do modelo com a aplicação web

    Interface simples para entrada de dados e exibição do resultado

    Repositório público no GitHub

4. Estrutura Esperada do Projeto

O repositório deve estar organizado de forma clara, contendo, no mínimo:

    Código de pré-processamento dos dados

    Código de treinamento do modelo

    Código de avaliação do modelo

    Arquivo do modelo salvo

    Código da aplicação web

    Arquivo README.md com explicações do projeto

5. README.md (Obrigatório)

O repositório deve conter um arquivo README.md explicando:

    Objetivo do projeto

    Descrição do dataset utilizado

    Tipo de modelo escolhido e justificativa

    Métricas utilizadas para avaliação

    Como executar o projeto localmente

    Como utilizar a aplicação web

6. Critérios de Avaliação

A avaliação será baseada nos seguintes critérios:

    Correta definição do problema de IA

    Qualidade do pré-processamento dos dados

    Escolha adequada do modelo

    Uso correto de métricas de avaliação

    Funcionamento do modelo salvo e carregado

    Integração correta entre modelo e aplicação web

    Organização e clareza do código

    Clareza e completude do README.md
